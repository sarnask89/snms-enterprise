# Auto-Import And Customer Expansion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement session-level network discovery auto-import with fallback customers and extend the customer domain to support full individual/company operator data.

**Architecture:** Extend the existing SQLite runtime schema with additive migrations, keep one `customers` table with a `customerType` discriminator, and add session-level auto-import logic behind a dedicated endpoint. Keep correspondence address on `customers` and installation address on `customer_devices`, while exposing filters and actions in the Nuxt UI.

**Tech Stack:** TypeScript, Express, TypeORM, SQLite, Nuxt UI, Node test runner
