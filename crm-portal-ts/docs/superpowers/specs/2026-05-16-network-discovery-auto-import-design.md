## Network Discovery Auto-Import Design

### Goal
- Add an explicit session-level auto-import action for discovery results.
- Parse Mikrotik lease comments when possible.
- Create fallback technical customers when parsing fails.
- Mark fallback customers in the customer table and allow filtering them in the customers UI.
- Optionally create tariffs and subscriptions from Mikrotik `rate-limit` values during auto-import.

### Scope
- New backend endpoint: `POST /api/v1/network-discovery/sessions/:id/auto-import`
- Customer data model extension with auto-generated markers
- Auto-import logic for `mikrotik_lease` and `dasan_onu`
- Optional tariff/subscription creation for Mikrotik leases
- Customers API filtering and Nuxt customers page filter
- Operations page action for session auto-import

### Import Rules
- `mikrotik_lease`
  - Read parsed comment from discovery payload.
  - If parsing succeeds:
    - create or reuse a customer derived from parsed fields,
    - create or reuse a customer device for the lease.
  - If parsing fails:
    - create a technical customer named `Nieznany Auto <id>`,
    - mark it as auto-generated,
    - create a customer device for the lease.
- `dasan_onu`
  - If no customer mapping exists, create a technical auto-generated customer and import a customer device.
- `dasan_mac` and `mikrotik_network`
  - Skip during customer auto-import and report as skipped.

### Customer Marking
- Add `isAutoGenerated` boolean on customers.
- Add `autoImportSource` string on customers.
- Surface both in customer serialization.
- Customers page gets:
  - auto-generated badge/column,
  - filter for all / auto-generated / normal customers.

### Tariff And Subscription Import
- Auto-import action accepts a flag to enable tariff/subscription import.
- For Mikrotik leases with `rate-limit`:
  - normalize `rate-limit`,
  - map identical normalized values to existing tariffs,
  - create a new technical tariff when no match exists,
  - create or reuse an active subscription linked to the imported customer and device.
- Auto-created tariffs use deterministic names and carry a technical description.

### Verification
- Migration test covers the new customer columns.
- Dedicated backend test covers:
  - parsed import,
  - fallback import,
  - auto-generated customer filtering,
  - tariff and subscription creation from `rate-limit`.
