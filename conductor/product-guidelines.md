# Product Guidelines - CRM Portal

## Voice & Tone
- **Professional & Efficient:** The tone should be business-like, focusing on clarity and speed.
- **Helpful:** Error messages and tooltips should provide clear, actionable guidance.
- **Direct:** Avoid marketing fluff; focus on data and functionality.

## UX Principles
- **Responsiveness:** Use Tailwind CSS to ensure the interface works seamlessly on desktops and tablets.
- **HTMX Interactions:** Prioritize partial page updates for a fast, "app-like" feel without full page reloads.
- **Data Density:** Present information clearly, using tables and cards to manage high volumes of data without overwhelming the user.
- **Feedback Loops:** Provide immediate visual feedback for all user actions (e.g., loading states, success messages).

## Branding & Aesthetics
- **Color Palette:** Professional blues and grays for stability, with distinct colors for alerts (red), warnings (amber), and success (green).
- **Typography:** Clear, sans-serif fonts optimized for readability in data-heavy environments.
- **Consistency:** Use consistent button styles, form inputs, and layout patterns across all modules.

## Technical Writing
- **Language:** The primary language for the user interface should be Polish, aligning with the target ISP operations and TERYT integration.
- **Terminology:** Use standard ISP and CRM terminology consistently.
